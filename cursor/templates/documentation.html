Project Overview

CipherStation is an enterprise-grade encryption platform built as my final project for CS50x. It combines military-grade cryptographic algorithms with a user-friendly web interface and secure message relay system. The project demonstrates advanced concepts in cryptography, web development, and secure software design.

Military-Grade Encryption

Supports AES-128, AES-192, AES-256, and ChaCha20-Poly1305 algorithms with Argon2id key derivation for maximum security.

Secure Message Relay

Ticket-based message system with automatic 24-hour cleanup for privacy and data protection compliance.

File Encryption

Encrypt and decrypt files of any type with automatic key generation and secure key management.

CLI Interface

Command-line interface for automation and integration with existing workflows and scripts.

Security First

Built with security best practices including secure key derivation, salt generation, and memory-safe operations.

Responsive Design

Modern, responsive web interface that works seamlessly across desktop, tablet, and mobile devices.

System Architecture

Core Components

  • ciphercore.py - Core cryptographic functions and algorithms
  • app.py - Flask web application and API endpoints
  • templates/ - HTML templates with responsive design
  • static/ - CSS, JavaScript, and static assets
  • tests/ - Comprehensive test suites for all functionality

Data Flow

  1. User enters message and password
  2. Password is converted to appropriate key length using Argon2id
  3. Message is encrypted using selected algorithm
  4. Encrypted data is stored with salt for key reconstruction
  5. Message can be sent to relay station or copied directly
  6. Decryption reverses the process using the same password

Security Features

Key Management

  • Argon2id Key Derivation: Industry-standard password-based key derivation
  • Salt Generation: Unique salt for each encryption operation
  • Key Length Validation: Automatic key length adjustment for different algorithms
  • Memory Safety: Secure memory handling and cleanup

Data Protection

  • 24-Hour Retention: Messages automatically deleted after 24 hours
  • Auto-Cleanup: Background thread removes expired messages
  • Capacity Management: Prevents memory overflow with 10,000 message limit
  • Secure Storage: Encrypted data never stored in plain text

Cryptographic Standards

  • AES-GCM: Authenticated encryption with associated data
  • ChaCha20-Poly1305: High-performance authenticated encryption
  • Nonce/IV Management: Cryptographically secure random generation
  • Algorithm Validation: Strict algorithm and parameter validation

Message Decryption

File Encryption

POST /encrypt Content-Type: multipart/form-data file: [binary file data] algorithm: "aes256" description: "Optional description" key: [optional key file]
